파일을 분리하여 javascript 코드를 외부에서 관리한다.
파일 분리의 장점: 새로운 파일을 만들면 모든 코드를 카피할 필요없이 해당 파일만 가져오면 된다.
해당 파일을 수정하면 모든 페이지에 변화가 반영된다.
한번 깔아둔 파일은 다음에는 네트워크를 통하지않아도 되므로 서버는 비용 절감할 수 있고 사용자도 네트워크 트래픽 절감 및 속도 빠름

library vs framework
library: 프로그램에 필요한 부품들이 정돈되어 있는, 재사용하기 쉽도록 되어있는 소프트웨어
framework: 게임, 웹 등 만들고자하는 것이 무엇이냐에 따라서 그것을 만들고자할 때,
framework를 기준으로 두고 쉽게 작업을 한다.
javascript의 library: jQuery 를 사용하면 생산성 향상
jquery를 직접 다운받아도 되고(사용료o) cdn을 통해(따로 서버에서 관리하는 것을 가져와 사용료x) 가져와서 사용해도 된다.
많은 종류의 라이브러리를 알수록 많은 기능을 구현할 수 있는 것

UI vs API
UI: 사용자가 시스템을 제어하기 위해서 사용하는 조작 장치
"User Interface로 유저가 특정 조작장치를 사용해 시스템에 관여하는 것을 의미"
API: alert와 같은 함수를 실행하면 경고창을 띄워주겠다는 javascript의 사용설명서를 통해 약속
-> 함수 alert는 경고창을 실행하는 조작장치로 일반인은 사용하지 않는다. application을 만들기 위해서 프로그래밍을 할 때 사용하는 조작장치를 API라고 한다.
모든 프로그래밍언어에 공통적으로 적용되는 것. 모든 애플리케이션들은 API를 순서대로 실행하는 것을 통해 만들어진다.
"Application Programming Interface를 의미하며 프로그래밍 언어가 제공하는 기능을 사용하게 해주는 인터페이스"
